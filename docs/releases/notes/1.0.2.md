## Kopi v1.0.2 - Settings Reliability Update

**Kopi 1.0.2** fixes the remaining first-open regressions around the menu bar window actions and restores the native macOS Settings experience.

---

### Menu Bar
- `Open History` and `Settings...` now dispatch after the status menu closes so the requested window stays visible on the first attempt

### History Window
- The first explicit `Open History` action now survives the app's initial activation instead of flashing briefly and disappearing

### Settings
- The menu bar `Settings...` action now reuses the app's real SwiftUI Settings scene instead of a plain fallback window
- The native macOS Settings toolbar style with tab icons is restored
- Settings reopening now prefers the app menu callback and only accepts selector fallbacks when a real settings window is actually present

### Quality
- Added regression coverage for first-activation window hiding, deferred status-menu actions, and native settings scene reopening
